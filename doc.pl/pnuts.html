Projekt Pnuts
Opis

Pnuts jest skryptowym językiem bazującym na JVM. Jego zadaniem jest umożliwienie systemom bazującym na platformie Java łatwego i wydajnego łączenia skryptów i kodu języka Java. Oto niektóre właściwości języka.

  • Prosta i czysta składnia
  • Interaktywny interpreter
  • Rozszerzalny system modułów
  • Elastyczne i dostosowywalne  API
  • Dynamiczna/statyczna kowersja do kodu bajtowego JVM
  • Jeden z najszybszych skryptowych języków implementowanych w Javie
  • Posiada wiele dobrych cech Javy (bezpieczeństwo, przenośność, itd.)

Strona domowa projektu to http://pnuts.dev.java.net/, gdzie można znaleść dokumentacje, kod źródłowy jak i najnowsze wydania dystrybucji.

Motywacja

Pnuts orginalnie został stworzony jako narzędzie testowania klas Javy w 1997 r. Od tego czasu język został znacznie rozbudowany którego zastosowanie skupia się na spełnieniau zadania bazującego na Javie skryptowego silnika zawierającego system modułów, kompilator kodu bajtowego i wiele więcej.


Od Tłumacza: najnowsza wersja języka zawiera wiele eleganckich i wygodnych elemetów jęzkowych i składniowych które dają nam możliwości znane z takich języków jak Python 

Od roku 2002 kiedy powstało skryptowanie WEB język Java posiadał już rozwiązanie JSP jednak jak wiadomo wielu webmasterów nie było programistami javy. Rozwiązanie bazujące na JSP wydawało się trudne dla nich do nauczenia i z tego powodu większość z nich skożystała z jęzka PHP i podobnych technologii. Powstał pomysł aby stworzyć skryptowe środowisko dla programistów nie będącymi programistami Javy dając im możliwości kożystania z rosnących możliwości platformy Java. 

Zalety
  1. Język został zaprojektowany specjalnie dla platformy Java
    Systemy skryptowe dla JVM poszły w dwuch kierunkach, jeden to tworzenie portów istniejących języków dla platformy Java (np. język Python dla javy to Jython), druga droga to tworzone nowe języki od początku.

    Systemy tworzone na podstawie istniejących języków mają jeden wspólny problem, utrzymywanie kompatybilności z istniejącymi systemami. Jeżeli istnieją różnice miedzy językim skryptowym a językiem bazowym muszą one pozostać np. jeżeli język skryptowy posiada inny system typów to dane przekazywane ze skryptu do kodu bazowego języka muszą być konwertowane. Na dodatek użytkownik musi uczyć się innej składni i zasad programowania w języku skryptowym, nie bez naczenia jest fakt że konwersje danych na połączniu śrokowiska skryptowego i języka bazowego mogą wpływać negatywnie na wydajność.

    Nowe języki projektowane z myślą o jezyku Java w tym Pnuts nie posiadają tych niedogodności, wszystkie wartości w Pnuts są prawdziwymi objektami języka Java więc zadna konwersja nie jest konieczna.

  2. Modularne API
    API Pnuts pozwala na proste łączenie skryptów z językiem Java. Wiele innych systemów jest o wiele bardziej skomplikowanych w tej dziedzinie.
  3. Bogaty zestaw standardowych modułów
    Pnuts dostarcza gotowe biblioteki funkcji jako standardowe moduły . Funkcje te pozwalają rozwijać aplikację dużo ławiej niż rozwiązania oparte tylko o możliwości Javy.  W dodatku , Pnuts dostarcza wyrafinowanego systemu modułów które użykownik może dodawać na potrzeby danej aplikacji . 
  4. Wysoka wydajność
    Pnuts jest jednym z najszybszych języków bazujących na platformie Java. Szybkość jest ważna wtedy gdy silnik skryptowy jest częścią rozbudowanej aplikacji ponieważ skrypty są często wąskim gardłem w wydajności takiej aplikacjii.
