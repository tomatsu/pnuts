Dostosowywanie interpretera

Skrypty Pnuts zazwyczaj są kompilowane do kodu bajtowego w czasie wykonania. Sposobem zmiany tego zachowania jest ustawienie atrybut (pnutsImpl) obiektu Context. Przykładowo skrypty mogą być wykonywane bez kompilacji lub z kompilacją i keszowaniem.

pnuts.lang.Implementation Object

pnuts.lang.Implementation definiuje interfejs implementacji interpretera. Instancja implementacji tego interfejsu jest cześcią obiektu klasy Context. Para metod Context.setImplementation() i Context.getImplementation() są setram i geterem tej właściwości.

Predefiniowane implementacje

Następujące implementacje są cześcią dystrybucyj Pnuts.

pnuts.lang.PnutsImpl
Interpreter AST.
pnuts.compiler.CompilerPnutsImpl
Kompilator w locie, domyślna implementacja dla komend.
pnuts.ext.CachedPnutsImpl
Mieszana implementacja keszująca kompilowane lub parsowane skrypty to ponownego użycia. Domyślna dla klas  Pnuts Servlet.
pnuts.security.SecurePnutsImpl
Bezpieczna wersja wykonująca skrypty w oparciu Java2 Security API
pnuts.security.JAASPnutsImpl
Bezpieczna wersja wykonująca skrypty w oparciu JAAS.

Zobacz Podpowiedzi na temat wydajności w celu prawidłowego wyboru implementacji w zależności od potrzeb. 

import pnuts.lang.*;
import pnuts.ext.*;

Context context = new Context();
context.setImplementation(new CachedPnutsImpl());
Pnuts.loadFile(fileName, context);

SecurePnutsImpl i JAASPnutsImpl są adapterami innych implementacji z dodatkiem funkcjonalności zabezpieczeń. Zobacz Budowanie chronionego środowiska skryptowego aby dowiedzieć się więcej. 

context.setImplementation(new SecurePnutsImpl(new CompilerPnutsImpl()));
Pnuts.load(new URL("..."), context);

Właściwość pnuts.lang.defaultPnutsImpl 

W systemowych właściwościach właściwość pnuts.lang.defaultPnutsImpl jest sposobem na zmianę domyślnej implementacji którą należy ustawić przed pierwszym użyciem jakiejkolwiek klasy z API Pnuts.

Zanim SecurePnutsImpl i JAASPnutsImpl nie posiadały domyślnych konstruktorów ustawianie tej właściwości było sposobem na zmianę implementacji domyślne.

Aby zobaczyć jaka jest używana implementacja można wywołać metodę Context.getImplementation(). 

> getContext().getImplementation()
