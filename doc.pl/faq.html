1. Pytania ogólne

Czym jest Pnuts ?
Pnuts jest skryptowym językiem mającym dostęp do API Javy w sposób interaktywny. Pnuts może być włączony do Javy jako silnik skryptów oraz może być łatwo wywoływany z kodu Javy.

Czy wymagana jest znajomość programowania w Javie ?
To zależy od tego jak wykożystywać będziesz Pnuts. Jeżeli istniejące moduły spełniają Twoje wymagania nie potrzebujesz bezpośrdnio wywoływać API i nie potrzebujesz znać Javy. Jeżeli chcesz pisać własne moduły potrzebna będzie podstawowa znajomość języka Java.

Jakie są wymagania aby użuwać Pnuts ?
Pnuts działa na platformie Java2SE. Został przetestowany na następujących wersjach.
  • Sun JDK1.3.1
  • Sun J2SDK1.4.2
  • Sun J2SDK1.5 beta
  • IBM JDK1.3
  • IBM JDK1.4
  • Mac OS X JDK1.4.2
  • Działa na SUN J2SDK1.6 beta2

Czy mogę użuwać Pnuts w komercyjnych produktach ?
Jasne. Pod warunkami licencji Warunki Licencyjne  można dystrybułować Pnuts.

2. Pytania techniczne

Jakie są zalety wględem innych języków skryptowych?
vs. jęzki oparte o inne skryptowe języki nie oparte na Javie
Pnuts kożysta z wielu możliwosci Javy jak wszechstronność API, automatyczne zwalnianie pamięci oraz możliwości maszyny wirtualnej jako takiej. Od czasu gdy Pnuts jest napisany w Javie działa na każdej platformie wspieranej przez Javę.

Istniejące syetemy języków skryptowych przenoszone do Javy muszą duplikować modele obiektowe z powodu zachowania kompatybilnosci. Kiedy taki język zostanie użyty w aplikacji programista musi operować na typach javy i opakowujących je typach rodzimych danego języka co może powodować spadek wydajności oraz czytelności samego programu.

vs. inne języki skryptowe bazujące na Javie
Największą zaletą jest rozszeżalny system modułów umożliwiający pisanie kodu na wyższym poziomie abstrakcji niż w Javie.  Z system modułów uzytkownik może użyć  potrzebnych modułów standardowych jak i swoich własnych jak zajdzie potrzeba.
vs. w porównaniu do czystego kodu Java
Pnuts zawiera duży zasób modułów pozwalających programować na wyższym poziomie.  Użytkownik może używać funkconalności Javy bez uzycia samego API Javy bezpośrdnio. Dla programistów Javy Pnuts może służyć jako narzędzie redukujące poziom komplikacji systemów przez kombinację obu języków w jednej aplikacji.
Jak szybko działa Pnuts?
Pnuts jest jednym z  najszybszych silników skryptowych bazujących na Javie. Wydajność Pnuts jest zbliżona lub nawet lepsza niż języków przeniesionych do Javy tj. Perl, Lua, itd. Tu znajdziemy wyniki testów wydajnosci Strona domowaPnuts.
Czemu jest inna składnia niż w Javie ?
W Pnuts "class classname" oznacza klase a w Javie zwyczajne "classname".
Tu jest problem kiedy zdefiniujemy globalną zmienną "java" co powoduje że podpakiety i klasy z pakietu "java" nie mogą byc prawidłowo odczytane. Z taką składnią odróżniamy nazwy klas od instancji zmiennych, problem można uznać za rozwiązany.

Pnuts uzywa "::" w dostępie do składników statycznych.
Pnuts nie posiada typów zniennych oraz nie ma różnicy między klasą a typem oraz klasa o obiektem Class. Nowa składnia "::" pozwala na dostęp do klasy jako objektu.

Czy mogę skonwertować skrypty Pnuts do kodu żródłowego lub kodu bajtowego Java ?
Pnuts działa domyślnie z kompilacją w locie do kodu bajtowego. API pozwala na kompilację skryptu do normalnego pliku Java class.

Jak mogę definiować klasy Javy ?
  • W wersji 1.1, wyrażenie 'new' pozwala na dziedziczenie klasy oraz kreację instancji tej klasy.
  • funkcja classGenerator() zdefiniowana w module   pnuts.lib   pozwala na definiowanie klas Javy przez mapowanie funkcji na metody lub definiowanie właściwosci JavaBeans.
  • moduł jdtc pozwala na kompilację prawdziwego kodu Javy.
  • Od wersji 1.2 Pnuts posiada możliwość definiowania klas, zobacz Definiowanie klas

Jak mogę używać wyrażeń regularnych podobnie jak w Perl ?
moduł pnuts.regex posiada funkcję obsługujące wyrażenia regularne.
