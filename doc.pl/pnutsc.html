Kompilacja skryptów z linii poleceń

pnutsc komendy

C:\> pnutsc { -d katalog | -o wyjściowy_plik_jar } { -v } { -main klasaMain { -m nazwaModułu } } { -prefix nazwa }{ -no_proxy } { { -C katalog_bazowy } plik_skryptu | wejściowe_pliki_jar } ...

pnutsc kompiluje podany plik plik_skryptu lub pliki wejściowe_pliki_jar.

opcja -o wyjściowy_plik_jar powoduje zapisanie skompilowanych klas do pliku JAR output_jar_file, gdy plik JAR instnieje klasy zostaną do niego dopisane.

gdy brak opcji -o klasy zostanie zapisane w katalogu z opcji -d katalog lub gdy jej brak w bieżącym katalogu

-v włącza opcje gadatliwą, wypisywane są informacje z przebiegu kompilacji.

jeżeli jest podane -no+proxy wygenerowany kod nie jest zależny od pakietu pnuts.compiler jednak wywołanie metod nie bezie zoptymalizowane.

-prefix name  ustawia domyślny prefiks wygenerowanych klas 
D:\pnuts> pnutsc -o pnuts-precompiled.jar pnuts/util/*.pnut pnuts/lib/*.pnut pnuts/gui/*.pnut pnuts/regex/*pnut
-C ustawia bieżący katalog na katalog_bazowy a następnie kompiluje

If -main powoduje wygenerowanie klasy startowej mainClass z metodą main() w której jest wykonany kod. 


public class mainClass extends pnuts.lang.Runtime {
   public Object run(Context ctx){
      ctx.usePackage(module1);
      ctx.usePackage(module2);
      ...
      new className1().run((Context)ctx.clone());
      new className2().run((Context)ctx.clone());
      ...
   }
   public static void main(String args[]){
      Pnuts.set("$args", args);
      new mainClass().run(new Context());
   }
 }
Przykład: hello.pnut 
println("hello")
C:\> pnutsc -o hello.jar -main Hello -m pnuts.lib hello.pnut
C:\> java -classpath "hello.jar;pnuts.jar;pnuts-modules.jar" Hello
