Kompilator i Interpreter AST

Kompilator Pnuts kompiluje skrypty w czasie wykonania i generuje w pamięci kod bajtowy, jest to domyślne zachowanie interpretera które można zmienić za pomocą komendy -pure która wymusza interpretacje drzewa AST. Wykonywanie skompilowanych skryptów jest dużo wydajniejsze niż interpretacje AST.

Opcje linii poleceń

C:\> pnuts { -pure } { skrypty }

Gdy opcja "-pure" jest podana wyłączany jest system kompilacji w locie i użyty zostaje interpreter AST. 

C:\> pnuts
Pnuts version 1.0rc2 (04/16/2002), 1.3.1_02 (Sun Microsystems Inc.)
Java HotSpot(TM) Client VM (mixed mode)
> getContext().getImplementation()
pnuts.compiler.CompilerPnutsImpl@df503
C:\> pnuts -O { skrypty }

-O powoduje optymalizację kompilacji - numery linii błędów nie są dostępne.
