Zezwolenia pakietu

Pnuts API pozwala chronic pakiety  Pnuts packages przez niezaufanym kodem używając  Java2 security API.

klasa pnuts.security.SecurePackage

pnuts.security.SecurePackage jest podklasą pnuts.lang.Package. Kiedy ustawiany jest SecurityManager potrzebne jest ustawianie  pnuts.security.PackagePermission aby dodać/usunąć pakiet lub zmienić wartość zmiennej pakietu.

Aby klasa  To make pnuts.security.SecurePackage była domślną klasą pakietu należy zdefiniować właściwość pnuts.package.factory jako pnuts.security.SecurePackageFactory przed inicjalizacją klasy pnuts.lang.Package. 

pnuts -J-Djava.security.manager -J-Dpnuts.package.factory=pnuts.security.SecurePackageFactory

klasa pnuts.security.PackagePermission 

public pnuts.security.PackagePermission(String pakiet , String akcje)

Parametr  pakiet dopuszcza aby nazwa pakietu którą kontroluje mogła była okręslana za pomcą znaków maskujących "*" na końcu nazwy, mozliwe jest też użycie znaków  "::", "." lub po prostu nazwy bezposredniej.

Drugi parametr  actions określa liste akcje oddzielonych przecinkiem.

write
zezwala na modyfikacje pakietów za wyjątkiem pakietów anonimowych.
add
zezwala na rejestrację pakietów.
remove
zezwala na wyrejestrowanie pakietów.

Przykładowo skrypt  http://javacenter.sun.co.jp/pnuts/examples/awt/calculator.pnut stara się zarejestrować nowy pakiet o nazwie  "calculator" oraz zdefiniwać w nim funkcje. Aby bezpiecznie wykonać skrypt na platformie  Java2 polecenie PackagePermission("calculator", "write, add") powinno pozwolić na taką operację.

Oto przykład pliku zezwoleń wymaganych przez ten skrypt. 

grant codebase "http://javacenter.sun.co.jp/pnuts/examples/-" {
      permission pnuts.security.PackagePermission "*", "write, add";
};

Zdalne skrypty działają w chronionym kontekście i są determiniowane przez adres  URL na zasadzie analogicznej do apletów. Domyślnie skrypty nie mają dostępu do lokalnego systemu plików i systemowych właściwości. Za pomoca pliku pozwoleń użytkownik może zmienić te zasady

Kiedy skrypt zdalny jest wykonywany jako anonimowy pakiet, skrypt może definiować zmienne globalne gdyż nie istnieją zasady dostępu dla aninimowych pakietów.
