Wczytywanie skryptów

Wczytywanie z plików lokalnych

loadFile ( String nazwaPliku {, Context kontekst } )

loadFile() wczytuje i wykonuje skrypt z podanego pliku fileName. Zwraca wynik ostatniego wykonanego wyrażenia.

np. (Windows) 
loadFile("..\\examples\\misc\\compose.pnut")
loadFile(`..\examples\misc\compose.pnut`)
loadFile("../examples/misc/compose.pnut")
np. (Unix) 
loadFile("/home/pnuts/sample.pnut")

Wczytywanie przy użyciu ClassLoader'a

load ( String skrypt {, Context kontekst} ) lub
( URL skrypt {, Context kontekst } ) lub
require ( String skrypt )

load() wczytuje i ewaluuje  script zwracając wynik ostatniego wyrażenia. Jeżeli Context.getClassLoader() zwraca referencje do loadera to funkcja load() szuka skryptu za pomocą tego loadera. W przeciwnym przypadku użyty zostanie systemowy loader klas.

require() wczytuje script jeżeli nie został jeszcze wczytany.

Końcówka '.pnut' w nazwie skryptu może być pominięta przy load() i require().

Gdy końcówka '.pnut' zostanie pominięta interpreter najpierw próbuje załadować klasę o podanej nazwie jako skompilowanej wersji skryptu, gdy takiej nie ma wczytuje podany plik.. Zobacz "Implementacje ładowalnych klas w javie".

autoload ( String nazwa , String nazwaSkryptu)

autoload() łączy nazwę name z scriptName co powoduje w przypadku braku definicji automatyczne wczytanie skryptu skojarzonego z nazwą.

Końcówka '.pnut' w nazwie skryptu może być pominięta przy autoload().

np. 
autoload("ls", "pnuts/util/ls")
