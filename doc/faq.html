1. General Questions

What's Pnuts?
Pnuts is a script language which can access Java APIs interactively. Pnuts can be embeded in a Java program as a scripting engine and be called easily from Java.

Java Programming Skill is required?
It depends on how you use Pnuts. If existing modules provides all you need to do, you do not have to call Java API directly. In that case, you do not need Java knowledge at all. If you develop a new module or you want to do something that existing modules do not provide, you will need basic knowledge of Java API.

What are the requirements to use Pnuts?
Pnuts runs on J2SE. It has been tested on the following environments.
  • Sun JDK1.3.1
  • Sun J2SDK1.4.2
  • Sun J2SDK1.5 beta
  • IBM JDK1.3
  • IBM JDK1.4
  • Mac OS X JDK1.4.2

Can I use Pnuts in my commercial products?
Sure. Under License Term, you can distribute Pnuts with no royalty.

2. Technical Questions

What's an advantage over other languages?
vs. legacy scripting languages
Pnuts takes advantages of various Java functionalities, such as versatile APIs, garbage collection, and state-of-the-art virtual machine. Since Pnuts is written in Java, it runs on every platform that supports Java.

Existing scripting language systems ported to Java must have duplicated object models because they have to keep the compatibility. When such a scripting system is embedded into an application, the programmer has to wrap/unwrap the two types of objects, which degrade the readability and the performance at the same time.

vs. other Java-based scripting languages
The most notable feature of Pnuts is its extensible module system that enables one step higher level of programming than Java API programming. With the module system, user can pick the necessary modules for his/her application, or add their own modules for the application if needed.
vs. Java-only solution
Pnuts comes with various kinds of modules that allows higher level programming. User can utilize the functionalities of Java, not using Java API directly. It means that Pnuts can target broader user base. For experienced Java programmers, Pnuts can be used as a tool that reduces the complexity of the system by combining the two languages in one application.
How fast can Pnuts run?
Pnuts is one of the fastest Java-based scripting engine. The performance of Pnuts is competitive or even superior to legacy scripting systems, such as Perl, Lua, etc. There are some links to benchmark results in Pnuts Home.
Why use different syntax than Java ?
In Pnuts, "class classname" specifies a class, although just "classname" in Java.
There is a problem when a global variable "java" is defined, for example, classes in "java" package can not be accessed properly. With a syntax that clarify the border of a class name and an instance variable, the problem can be solved.

Pnuts uses "::" to access static memebers.
Pnuts does not have types for variables. And there is no distinction between a class as a type and a class as a Class object. The new syntax "::" allows to access a class as an ordinally object.

Can I convert a Pnuts script to Java bytecode?
Pnuts runs with on-the-fly bytecode compiler by default. Also, Pnuts API provides a way to compile a script to Java class files.

How do I create Java classes in a script?
  • On version 1.2, syntax for class definition can be used to define Java classes.
  • On version 1.1, 'new' expression allows you to extend a class and create an instance of the extended class.
  • classGenerator() function defined in pnuts.lib module allows you to define Java classes by mapping a function to a method or specifying JavaBeans properties.
  • jdtc module provides functions to compile true Java code.

How do I use regular expression like in Perl?
pnuts.regex module provides functions that handles regular expressions.
