Executing in the Event-Dispatcher Thread

useEventThread( boolean flag )

If flag is true, subsequent scripts are executed in pnuts.awt.EventQueueConfiguration, i.e. method/constructor calls and field access in subsequent scripts are executed in the event dispatcher thread. If flag is false, the previous Configuration which was used when useEventThread(true) was used. 

useEventThread(true)
Thread::currentThread()    ==> Thread[AWT-EventQueue-0,6,main]
useEventThread(false)
Thread::currentThread()    ==> Thread[main,5,main]

Functions defined in EventQueueConfiguration are executed in the same configuration even if it is called from somewhere in a different configuration.
